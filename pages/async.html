ASyncrounous Methods

Android like we have mentioned uses a single thread on a main event loop. This brings up some obvious issues expecially when you need to have long data processing. Thankfully android provieds some very good multithreading frameworks to allow for async method calls.

Java Executors and Thread Pools

The first main thing that android provides is everything that the java libraryies provide. A common thing to do is to create an executor or a thread pool withen the service. Then any intent you get can be passed off to the executor or thread pool to execute.


public class MyService extends Service {

    private Executor executor = Executors.newSingleThreadExecutor();


    public MyService() {

    }

    @Override
    public void onCreate() {
        super.onCreate();

    @Override

            }
        });
        return super.onStartCommand(intent, flags, startId);
    }

    @Override
    public void onDestroy() {
        super.onDestroy();
    }

    @Override
    public IBinder onBind(Intent intent) {

    }
}

Intent Service

The next option is what android calls the intent service. Services do not process their intents on a seperate thread and because google didnt want everyone to have to create their own executors they created their own version of what we just wrote called a IntentService. The Intent Service will process all intents on a seperate thread just like we did. To do this we create an intent service that has the framework.
